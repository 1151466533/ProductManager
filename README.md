# 关于产品

### 碎片

产品的构建和设计是交叉学科的融合艺术，需要跨界的协作。

有非常多不同的产品构建思路，有强调敏捷的、强调核心需求的、强调设计方法的、强调体验的，等等，无论你采用了哪一种，它都是过时的；而产品理念始终是引领趋势的，理念的成型大多是半自觉的，它与功能的设计、业务的流程并没有关系，它来源于生活，关于美学、艺术、自然、人文、商业、科技、哲学、等等。

对产品的理解应该是广义的，生活中大多数的物件都是产品，它们都关于时间、关于熵。

好的产品应该总是能为用户带来增益价值的、苛求细节的、追求革新的、美观的、极简的、易用的。

产品经理并不是一个特别好的描述，我更喜欢用PA(Product Artist)来表达所做的事情。

不太相信有优秀的职业产品经理。

### 产品设计流程

待拓展

- **内核**
    1. 理念层
        1. 认知
        2. 理念
        3. 原则
        4. 设计语言
    2. 逻辑层
- **底层**
    1. 概念层
        1. 灵感
        2. 新解决方案/找到缺口
        3. 可行性研究
        4. 构建和验证概念原型
    2. 设计层
        1. 商业模型
        2. 美学模型
        3. 逻辑模型
        4. 结构模型
- **上层**
    1. 协同开发
    2. 测试和验证
    3. 保持极简
    4. 迭代

方法、流程、规范会加快效率，但同样会抑制灵感，无论在任何环节，都需要否定。

排斥冗杂、低耦合。

保持极简、轻、弹性。

依赖逻辑，排斥逻辑。

好的产品一般不会脱胎于规范。

> Innovation is saying no to a thousand things.   - Steve
